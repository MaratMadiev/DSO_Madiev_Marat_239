# ADR-001: Стратегия безопасности MVP - Приоритизация контролей

## Context
Для MVP Suggestion Box необходимо определить, какие контроли защиты реализовать в первую очередь. Система обрабатывает конфиденциальные пользовательские данные (предложения, учетные данные) и требует баланса между безопасностью и скоростью разработки.

## Decision
Реализовать security по принципу "защита самых критичных активов". В MVP фокусируемся на:
1. **Защита учетных данных** - предотвращение утечки паролей
2. **Изоляция данных** - пользователи видят только свои данные
3. **Базовое шифрование** - защита данных в transit

Отложить до Release 1: продвинутый мониторинг, защиту от DDoS, security scanning.

## Alternatives
- **Full Security**: Все 8 NFR сразу - слишком медленно
- **Minimal Security**: Только аутентификация - риск утечек

## Consequences
**Positive**: Быстрый запуск, защита критичных данных, ясный roadmap
**Negative**: Временная уязвимость к DDoS, ограниченный аудит

## Security Impact
- **Покрытие рисков**: R1, R2, R5 (критические)
- **Непокрытые риски**: R3, R7 (средние/низкие)
- **STRIDE**: Spoofing, Tampering, Information Disclosure - покрыты

## Rollout Plan
1. SEC-NFR-002 (пароли) + SEC-NFR-006 (TLS)
2. SEC-NFR-001 (аутентификация) 
3. SEC-NFR-003 (авторизация) + интеграция
