# Политика безопасности

## Сообщение об уязвимостях

**Пожалуйста, НЕ сообщайте об уязвимостях через публичные GitHub issues.**

Если вы обнаружили проблему безопасности в этом проекте, сообщите об этом приватно:
- **Преподаватель**: [Andrey Lednev] - [@aNd1CE]

### Что включить в отчёт:
- Описание уязвимости
- Шаги для воспроизведения
- Потенциальный impact
- Предлагаемое исправление (если есть)

## Практики безопасности

### Конфиденциальные данные
- **Ни при каких обстоятельствах** не хранить API-ключи, пароли или секреты в репозитории
- Использовать переменные окружения для конфигурации
- Во время курса использовать **только синтетические данные** (без ПДн/платежных данных)

### Безопасность кода
- Используйте единый формат обработки ошибок (см. README)
- Валидируйте все входные данные
- Следуйте принципу наименьших привилегий

### Тестирование безопасности
- Регулярное обновление зависимостей
- Проверка безопасности в CI-пайплайне
- Code review обязателен для всех изменений

## Область применения

Данная политика безопасности применяется к:
- Всему коду в этом репозитории
- Зависимостям из requirements.txt
- GitHub Actions workflows
